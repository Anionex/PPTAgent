## 已完成

### 1. 修复图片裁剪变形 (e6c3b14)
- **问题**: `Presentation.from_file()` → `save()` 后，source.pptx 中图片裁剪比例失真（如 logo 高度被压缩一半）
- **原因**: `ImageElement.build()` 中 `dict_to_object(self.style["img_style"], shape.image)` 把裁剪值写到了 `Image` 对象上（无 crop 属性，静默失败），应该写到 `shape` 上
- **修复**: `shapes.py:894` — `shape.image` → `shape`

### 2. 修复组合形状子元素飞到左上角 (4907f66)
- **问题**: 组合形状（GroupShape）的子元素在标准化后全部飞到页面左上角 (0,0)
- **原因**: `ShapeElement.build()` 通过 `deepcopy(self.sp)` 插入原始 XML（带组合内局部坐标），然后 `fill.build()` 会替换整个 `spPr` 元素（包含 `xfrm` 坐标），导致之前设置的 `shape_bounds` 被覆盖
- **修复**: 将 `dict_to_object(shape_bounds)` 移到 `fill.build()` 和 `line.build()` 之后执行

### 3. 修复嵌套组合形状坐标转换 (9d070ac)
- **问题**: 嵌套组合（如 组合3 → 组合33 → Text2 "AI COMMERCIALIZATION INNOVATION"）的子元素仍然飞到左上角
- **原因**: `parse_groupshape()` 只转换一层坐标。嵌套组合的 `__post_init__` 在父组合更新坐标之前就已执行，导致后代元素留在错误的坐标空间
- **修复**: 新增 `_apply_group_bounds()`, `_remap_bounds()`, `_remap_descendants()` 三个方法，递归地对嵌套组合执行仿射坐标变换。同时修复了原代码用 `self.data[idx]` 索引但跳过不可见 shape 后索引不对齐的 bug

### 4. 修复结构化输出 JSON 解析失败 (9291a30)
- **问题**: 通过 aihubmix 代理调用 Claude/Gemini 时，OpenAI SDK 的 `parse()` 方法失败，因为模型返回 ` ```json...``` ` 包裹的 JSON
- **原因**: API 代理不完全支持 OpenAI 的结构化输出协议，模型响应带 markdown 代码块
- **修复**: `llms.py` 中将 `parse()` 替换为 `create()` + `response_format={"type":"json_object"}`，然后用 `get_json_from_response()` 清理 markdown 包裹，再用 Pydantic `model_validate()` 验证
